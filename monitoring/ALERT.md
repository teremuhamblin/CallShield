# Alerts

Configuration des alertes pour le monitoring de CallShield.

## Objectifs
- Détecter les comportements anormaux
- Surveiller les performances critiques
- Alerter en cas de défaillance ou surcharge
- Assurer la disponibilité du service

## Types d’alertes
- Erreurs système
- Temps de réponse élevé
- Taux d’échec anormal
- Ressources saturées (CPU, RAM, I/O)
- Exporteurs inaccessibles

## Intégration
Compatible avec :
- Prometheus Alertmanager
- Grafana Alerting
- Webhooks externes
